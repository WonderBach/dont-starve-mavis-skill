# App Agent 自动检测优先工作流评估

## 结论

这个方案比长问卷更适合做 Mavis Skill。

推荐主流程：

1. 用户触发 Skill 后，APP Agent 默认先检测 Steam。
2. APP Agent 检测 Steam 库里是否安装 Don't Starve 单机版或 Don't Starve Together 联机版。
3. 根据检测结果走单机/联机不同 UI 路径，进入停尸房/讣告并截图。
4. Mavis 本体读取截图，OCR 出 `Days Lived` 和 `Cause of Death`。
5. 饥荒 Skill 根据天数和死因做推理。
6. 如果用户选择介入，再输出攻略、物资代码或属性代码。

上传截图和手动询问资源状态都作为兜底：未检测到 Steam/游戏、非 Steam 版本、自动化路径失败、截图不是停尸房、OCR 读不清、或用户不愿意自动操作时才启用。

## 工作量评估

难度：中等偏低。

如果 Mavis 已经有图片理解能力，Skill 侧不需要自己做 OCR 模型，只需要写清楚输入输出契约和推理规则。主要工作量在三处：

- 自动化链路：APP Agent 能否稳定打开游戏、进入停尸房、截图。
- 视觉抽取：Mavis 能否稳定读出表格里的 `Days Lived` 和 `Cause of Death`。
- 规则推理：把死因映射到生存链，并输出下一局方案。

相对之前的长问卷方案，自动检测优先的优势是：

- 用户负担更低，用户点开 Skill 后系统先自己尝试检测和截图。
- 数据更准，天数和死因不用靠用户回忆。
- Agent 更像“自动诊断”，产品感更强。

风险是：

- 游戏界面语言、分辨率、字体、遮挡会影响 OCR。
- 停尸房可能有多条记录，需要决定分析哪一条。
- `Cause of Death` 是最后一击，不等于根因，不能只机械翻译。

## 三层分工

### APP Agent

负责动作，不负责推理。

它应该做：

- 打开或切回《饥荒》。
- 进入 Morgue / 停尸房 / 死亡记录界面。
- 截图并把图片交给 Mavis。

它不需要理解“为什么死”，也不需要判断下一局怎么改。

### Mavis 本体

负责读图和 OCR。

它应该输出结构化字段：

```json
{
  "days_lived": 23,
  "deceased": "Wilson",
  "cause_of_death": "Darkness",
  "mode": "Survival",
  "ocr_confidence": 0.92,
  "raw_ocr_text": "..."
}
```

最低成功条件是读出：

- `days_lived`
- `cause_of_death`

读不出 `deceased` 或 `mode` 可以降置信，但不一定阻塞。

### 饥荒 Skill

负责推理和输出。

它应该做：

- 先复述读图结果：你活了多少天，停尸房显示死因是什么。
- 用天数判断阶段或季节，但只作为辅助判断。
- 用死因映射根因链。
- 给下一局最小修正方案。
- 根据用户介入偏好，决定是否给控制台代码。

## 天数和季节判断

如果确认是 RoG/Giant Edition 秋季起始默认世界，可以粗略用：

- 第 1-20 天：秋季建设窗口
- 第 21-36 天：冬季/首冬窗口
- 第 37-56 天：春季雨湿窗口
- 第 57-72 天：夏季过热/防火窗口

但这条规则必须降置信使用，因为：

- RoG 可能春季起始。
- DST 季节长度和单机不完全相同。
- 用户可能改世界设置。
- 裸 Don't Starve 本体不是四季制。

所以最终判断优先级应是：

1. 停尸房死因。
2. 天数窗口。
3. 用户文本或截图上下文。
4. 低置信时最多补问 1 题。

## 用户介入能力

这个功能应该保留，而且放在读图之后。

推荐三档：

- 只给攻略：默认档，只输出下一局路线和优先级。
- 给物资代码：用户想补资源，但仍保留探索和操作。
- 改人物属性/开挂：用户只想快速越过卡点。

输出时不要默认上代码。先给诊断，再根据用户选择展示对应援助。

## 最小输出模板

```text
我读到这条死亡记录：
- 生存天数：第 23 天
- 停尸房死因：Darkness
- 阶段判断：大概率处于首冬窗口

这不是单纯“血量不够”，更像是光源链断了：
冬季夜晚更长，移动光源或燃料不足，最后被黑暗/Charlie 结算。

下一局最低修正：
1. 第 10 天前做火坑和锅。
2. 第 15-20 天整理木头、草、树枝。
3. 出门至少带 2 根火把或等价材料。
4. 冬季夜晚不远行。

你想要我只给攻略，还是直接给物资代码/属性代码？
```

## 规则落点

当前数据库已对应调整：

- `data/morgue_ocr_rules.json`：自动检测优先、截图兜底、三层分工、OCR 字段、天数/死因推理。
- `data/agent_cards.json`：自动检测入口为主，截图上传和手动问答为兜底。
- `data/seasonal_survival.json`：四季生存链和季节死因映射。
