# 导师对齐说明：《饥荒》四季生存规划 MVP

## 1. MVP 口径建议

原文写“普通单机版为主”，但《饥荒》版本口径容易混淆：

- 裸本体 Don't Starve 是 Summer/Winter 两季，没有 Autumn。
- Reign of Giants 单机和 DST 才有 Autumn/Winter/Spring/Summer。
- 当前中文新手攻略语境里，“单机普通世界”经常默认指启用 RoG/Giant Edition 的单机体验。

建议产品文档改为：

> 首版以《饥荒》单机 RoG/Giant Edition 默认世界的一年四季基础生存为主，首冬作为冬季重点场景；兼容裸 DS 与 DST 的基础差异。

这样既符合玩家认知，也避免和 wiki 的严格版本定义冲突；同时能接住用户在春季、夏季或任意天数打开 Agent 的场景。

## 2. 四季时间线

主线场景以单机 RoG/Giant Edition 默认世界为主：

- 秋季：建设窗口，重点是科技、基地、食物、火源、基础战斗。
- 冬季：低温和长夜窗口，重点是暖石、保暖、燃料、冬季食物路线。
- 春季：雨季和潮湿窗口，重点是雨具、避雷、替代食物路线、理智压力。
- 夏季：过热和防火窗口，重点是冷火/冷暖石、防暑装备、自燃防护和食物保鲜。

兼容备注：

- DST 首冬第 21 天开始，到第 35 天结束，冬季 15 天。
- 裸 DS 本体是 Summer/Winter 两季，没有 Autumn/Spring；如果用户明确是裸本体，不应称为“秋季开局”或“春季雨季”。

## 3. 数据库边界

首版只覆盖“一年四季基础生存”直接相关的信息：

- 关键资源：火源、锅、陷阱、晾肉架、冰箱、基础护甲武器、暖石、雨具、避雷针、冷火/冷暖石、防暑和防火方案。
- 时间节点：开局建设、换季准备、已入季应急、反复季节死亡复盘。
- 核心威胁：冻结、断粮、黑暗/低理智、猎犬、鹿角怪、雨湿/雷击/青蛙雨、过热、自燃/火灾。

首版不做：

- 全角色差异化规划
- 完整食谱库
- 高级 Boss 击杀攻略
- 洞穴/远古/全 DLC
- DST 专属复杂机制

## 4. Agent 产品价值

新手死在季节变化里通常不是“完全不会玩”，而是没有形成“通用生存链 + 当季环境链”的闭环。

通用生存链：

- 食物链：断裂后饥饿值归零，最终转化为血量持续扣减，并迫使玩家冒险采集。
- 光源链：断裂后黑暗/夜间行动失败导致 Charlie 攻击、理智下降或意外战斗。
- 基础战斗安全链：猎犬、季节生物和暗影生物会把其他断链快速转化为死亡。
- 基地安全链：位置、路线、防火、避雷和巨人处理会放大或缓冲其他链。

当季环境链：

- 冬季保暖链：断裂后体温下降，最终转化为血量持续扣减。
- 春季雨湿链：断裂后潮湿、低理智、雷击、青蛙雨和食物路线变化叠加。
- 夏季降温/防火链：断裂后过热扣血，或基地和资源被自燃烧毁。

血量、饥饿、理智是 UI 上的实时结果；链状态才是可提前规划的原因。基地稳定、战斗装备和探索路线不再作为单纯展示项，而是作为放大或缓冲各条链的上下文。

因此 Agent 不应像百科一样堆知识，而应像教练一样先判断“当前季节最可能断哪条链”，排序优先级，并按用户愿意接受的干预程度输出攻略、物资代码或属性代码。

## 5. 停尸房截图解读

停尸房截图应作为主入口，而不是补充入口。处理流程应是：

1. APP Agent：打开或切回《饥荒》，进入停尸房/死亡记录界面并截图。
2. Mavis 本体：对截图做 OCR，抽取 Days Lived、Deceased、Cause of Death、Mode。
3. 饥荒 Skill：先输出生存天数和直接死因，再把直接死因反推到根因链，并给下一局最低修正方案。

没有截图、截图不是停尸房、OCR 读不清，或用户不允许自动操作时，才启用手动询问作为兜底。

注意：停尸房的 Cause of Death 是最后一击或直接死亡来源，不等于根因。例如 Terrorbeak/Crawling Horror 更可能说明光源链和理智恢复断裂；Hound/Deerclops 更像准备不足和场景处理问题；Overheating、Fire/Burning、Frog、Lightning 等则需要结合当前季节回推到降温链、防火链或雨湿链。

用户介入能力仍保留，但放在读图之后：默认只给攻略；用户选择后再展示物资代码或属性/开挂代码。
