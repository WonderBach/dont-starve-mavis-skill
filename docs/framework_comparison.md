# 首冬死亡逻辑框架对比

## 结论

采用新框架：

> 首冬生存能力 = 保暖链 × 食物链 × 光源链 是否同时成立。

旧框架“保暖能力 / 食物续航 / 光源掌控 / 基地稳定”适合做展示清单，但不适合作为规则层的根因模型。基地稳定应降级为上下文因素，因为它本身不直接导致死亡，而是让三条链更容易闭合或断裂。

## 旧框架的问题

旧框架把四个维度并列：

- 保暖能力
- 食物续航
- 光源掌控
- 基地稳定

问题是：基地稳定和前三个不是同一层级。玩家不会因为“基地稳定”这个抽象项直接死亡，而是因为基地位置/路线/结构安排让保暖链、食物链或光源链断掉。

## 新框架的优势

新框架更接近真实死亡因果：

- 保暖链断 -> 体温下降 -> 冻结 -> 血量持续扣减
- 食物链断 -> 饥饿归零 -> 血量持续扣减
- 光源链断 -> 黑暗/长夜 -> 理智下降或 Charlie 攻击 -> 暗影生物/黑暗伤害 -> 血量归零

这也解释了为什么 Agent 不应该优先问“你现在血量/饥饿/san 多少”。三维是实时结果，变化很快；链状态才是可提前规划、可稳定诊断的输入。

## 规则层采用方式

规则层只把三条链设为核心能力：

- 保暖链：御寒装备、火坑、暖石、便携火源、燃料。
- 食物链：锅、陷阱/兔子洞、稳定肉源/填充物、肉干/存粮。
- 光源链：固定光源、移动光源、备用燃料、夜间行动边界。

基地、战斗和探索距离作为上下文放大因子：

- 基地离兔子洞/牛群近，食物链更稳。
- 基地贴海岸或密林，企鹅/火灾/鹿角怪风险更高。
- 没有护甲武器，猎犬会把任何断链快速转化成死亡。
- 探图太远，保暖链和光源链更容易同时断。

## 对停尸房截图的意义

停尸房给的是“最后一击”，不是根因。

- Cause of Death = Freezing：高置信归因到保暖链断。
- Cause of Death = Hunger/Starving：归因到食物链断。
- Cause of Death = Darkness/Charlie：归因到光源链断。
- Cause of Death = Terrorbeak/Crawling Horror：直接死因是暗影生物，但根因通常是光源链和理智恢复不足。
- Cause of Death = Hound/Deerclops：归因到准备不足/上下文事故，再回查三条链。

所以图片 OCR 后的推理不能停在“你被某个怪打死了”，而要继续问：为什么你会在那个时间点、那个状态下被它打死？
